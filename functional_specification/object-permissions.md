# Permissions

The permission object defines particular activities that a [user](object-users.md) is allowed to perform if the permission belongs to a [role](object-roles.md) that is assigned to the user.

## User-defined fields

- Name
- Description

## Automatic fields

- Unique identifier

## Predefined entries

The permission table contains pre-defined entries to cover common cases:

- View users
- Edit users
- View project
- Edit project
- View sources
- Edit sources
- View qualities
- Edit qualities
- View tagging
- Perform tagging
- View queries
- Edit queries
- Execute queries
